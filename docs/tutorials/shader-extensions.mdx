# Shader Extensions

`ShaderExtension` packages reusable shader behavior that can be attached to a `Model` or `Computation`. This example renders a grid of triangles from one base shader pair while an extension adds procedural fill patterns such as hashes, checker strokes, and dot grids.

It is assumed you've set up your development environment as described in [Setup](/docs/tutorials).

The base shaders keep the vertex interface explicit:

- `fillPatternType` selects the procedural mask
- `fillPatternSize` controls stripe or dot spacing
- `fillPatternUv` gives the fragment shader local coordinates inside each triangle

That split matters for WebGPU. Vertex attributes are pipeline inputs, so the model still declares its `bufferLayout` and attributes directly. The extension supplies reusable shader behavior once those inputs are available.

Each backend-specific extension variant injects the pattern implementation into `fs:#decl`. The base fragment shader calls that implementation without knowing how the GLSL or WGSL source differs.

```typescript
import type {ShaderExtension} from '@luma.gl/shadertools';

const fillPatternExtension: ShaderExtension = {
  name: 'fill-pattern-extension',
  glsl: {
    injections: [
      {
        target: 'fs:#decl',
        injection: GLSL_FILL_PATTERN_SOURCE
      }
    ]
  },
  wgsl: {
    injections: [
      {
        target: 'fs:#decl',
        injection: WGSL_FILL_PATTERN_SOURCE
      }
    ]
  }
};
```

The injected shader source can expose a stable callable surface while still using language-specific syntax internally:

```typescript
const GLSL_FILL_PATTERN_SOURCE = /* glsl */ `
float extension_fillPatternStripeMask(float position, vec2 size) {
  float stepLength = max(size.x + size.y, 0.0001);
  float wrappedPosition = fract(position / stepLength) * stepLength;
  float edgeWidth = 0.0025;
  return 1.0 - smoothstep(size.x - edgeWidth, size.x + edgeWidth, wrappedPosition);
}

vec4 extension_applyFillPattern(vec4 color, float fillPatternType, vec2 uv, vec2 size) {
  float opacity =
    fillPatternType < 0.5 ? 1.0 : extension_fillPatternStripeMask(uv.x + uv.y, size);
  return vec4(color.rgb, color.a * opacity);
}
`;

const WGSL_FILL_PATTERN_SOURCE = /* wgsl */ `
fn extensionFillPatternStripeMask(position: f32, size: vec2<f32>) -> f32 {
  let stepLength = max(size.x + size.y, 0.0001);
  let wrappedPosition = fract(position / stepLength) * stepLength;
  let edgeWidth = 0.0025;
  return 1.0 - smoothstep(size.x - edgeWidth, size.x + edgeWidth, wrappedPosition);
}

fn extensionApplyFillPattern(
  color: vec4<f32>,
  fillPatternType: f32,
  uv: vec2<f32>,
  size: vec2<f32>
) -> vec4<f32> {
  let opacity =
    select(extensionFillPatternStripeMask(uv.x + uv.y, size), 1.0, fillPatternType < 0.5);
  return vec4<f32>(color.rgb, color.a * opacity);
}
`;
```

The model opts into the behavior through `extensions`, while the explicit attributes stay on the model:

```typescript
const model = new Model(device, {
  source: WGSL_SOURCE,
  vs: GLSL_VERTEX_SHADER,
  fs: GLSL_FRAGMENT_SHADER,
  extensions: [fillPatternExtension],
  bufferLayout: [
    {name: 'position', format: 'float32x2'},
    {name: 'fillPatternType', format: 'float32'},
    {name: 'fillPatternSize', format: 'float32x2'},
    {name: 'fillPatternUv', format: 'float32x2'}
  ],
  attributes: {
    position: positionBuffer,
    fillPatternType: fillPatternTypeBuffer,
    fillPatternSize: fillPatternSizeBuffer,
    fillPatternUv: fillPatternUvBuffer
  },
  vertexCount
});
```

The runnable example includes all nine pattern variants: `none`, `hash0`, `hash45`, `hash90`, `hash135`, `checker0`, `checker45`, `dotgrid`, and `dotgrid45`.

Use shared extension fields when a contribution is backend-neutral. Use `glsl` or `wgsl` blocks when the shader language needs different code, defines, or modules.
